¿Qué es la computación en la nube?

La computación en la nube es el acceso bajo demanda, a través de internet, a recursos informáticos —servidores, almacenamiento, bases de datos, redes, software— sin necesidad de poseerlos físicamente. Se paga solo por lo que se usa, y se puede escalar hacia arriba o hacia abajo en minutos.

Ejemplo concreto: Analizar la ENIGH 2022 (88,000 hogares) en tu laptop puede tardar minutos o fallar por falta de RAM. En Azure, puedes hacerlo en segundos con una VM de 64 GB que desaparece al terminar.
// modelos de servicio
IaaS · PaaS · SaaS
IaaS
Infrastructure as a Service
Tú gestionas: SO, aplicaciones y datos
  • Máquinas Virtuales de Azure
  • Redes virtuales (VNet)
  • Discos administrados
  • Control total del entorno
PaaS
Platform as a Service
Azure gestiona: infraestructura y SO
  • Azure PostgreSQL Flexible Server
  • Azure Databricks
  • Azure Machine Learning
  • Tú te enfocas en el código
SaaS
Software as a Service
Todo gestionado, listo para usar
  • Microsoft 365 / Teams
  • Power BI Service
  • GitHub (en la nube)
  • Sin instalación ni mantenimiento
// por qué importa
Beneficios para análisis de datos y política pública
Escalabilidad
Escala según demanda
Sube de 2 a 64 cores en 5 minutos para un análisis urgente. Baja de regreso cuando termines. Pagas solo las horas usadas.
Reproducibilidad
Entornos consistentes
Elimina el "en mi máquina sí funciona". Todos los integrantes del equipo trabajan en entornos idénticos configurados una sola vez.
Colaboración
Trabajo en equipo
Comparte datasets, notebooks y modelos con permisos granulares. Sin correos con archivos adjuntos, sin versiones perdidas.
Costo
Pago por uso
Sin inversión en servidores físicos. Un servidor de PostgreSQL cuesta ~$0.02/hora. Un análisis completo puede costar menos de $1 USD.

Componentes esenciales de Azure
Portal
Portal de Azure
Interfaz web para gestionar todos los recursos. Permite crear, configurar, monitorear y eliminar cualquier servicio sin escribir código.
portal.azure.com
Regiones
Regiones y centros de datos
Ubicación física de los servidores. Afecta latencia, costo y disponibilidad de servicios. Para México: East US o South Central US.
East US · South Central US · Brazil South
Suscripción
Suscripciones
Unidad de facturación y administración. Define límites de recursos disponibles. Una organización puede tener múltiples suscripciones por proyecto o equipo.
Subscription ID: xxxxxxxx-xxxx-xxxx
Resource Group
Resource Groups
Contenedor lógico para organizar recursos relacionados (VMs, Storage, bases de datos). Permite controlar permisos y costos por proyecto. Borrar el RG elimina todo lo que contiene.
rg-proyecto-enigh-2026
// primera práctica guiada
Lo que haremos hoy
1
Crear cuenta gratuita de Azure

Ir a azure.microsoft.com/free — incluye $200 USD de crédito por 30 días y servicios gratuitos por 12 meses. Requiere tarjeta de crédito para verificación (no se cobra).

2
Crear un Resource Group

Portal → Resource groups → + Create. Nombrar rg-curso-azure, seleccionar región East US. Este RG será el hogar de todos los recursos del curso.

3
Explorar servicios desde el portal

Navegar a All services y explorar categorías: Storage, Databases, Analytics, AI + Machine Learning. Identificar los servicios que usaremos los próximos 4 días.

4
Crear un Storage Account y un Blob Container

Portal → Storage accounts → + Create. Tier Standard LRS, habilitar hierarchical namespace para Data Lake Gen2. Crear contenedor mi-contenedor.
