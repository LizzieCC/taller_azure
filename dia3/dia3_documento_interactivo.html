Blob Storage
Ya lo conocemos · Refuerzo
Almacén de objetos para cualquier tipo de archivo. Lo creamos el Día 1. Hoy lo usamos como fuente de datos para el pipeline.
  • Archivos CSV, JSON, imágenes, modelos
  • Acceso con cadena de conexión o SAS token
  • SDK Python: azure-storage-blob
  • Acceso nativo desde Databricks (ABFS)
PostgreSQL + PostGIS
Flexible Server · Nuevo hoy
Base de datos relacional administrada. Activamos PostGIS para almacenar coordenadas y ejecutar consultas geoespaciales directamente en SQL.
  • PostgreSQL 16, totalmente administrado
  • PostGIS: tipo geometry, funciones ST_*
  • Firewall por IP desde el portal
  • Python: psycopg2 · Databricks: JDBC
Azure Databricks
Nuevo hoy · Análisis distribuido
Plataforma basada en Apache Spark. Conecta múltiples fuentes de datos en un entorno colaborativo de notebooks en la nube.
  • Workspace con notebooks colaborativos
  • Clusters de Spark administrados por Azure
  • Lectura directa de Blob sin descargar (ABFS)
  • Conexión a PostgreSQL con JDBC
// conceptos clave
Antes de entrar al portal
01
PostgreSQL Flexible Server vs. Single Server
+

Single Server es el modelo legado de Azure para PostgreSQL, actualmente en proceso de retiro. Flexible Server es el reemplazo actual: más configuración, mejor rendimiento y soporte de largo plazo.

  • Mantenimiento programado personalizable
  • Alta disponibilidad con zona de disponibilidad
  • Soporte completo de extensiones como PostGIS
  • Tier Burstable B1ms: el más económico, ideal para el curso
Costo estimado: el tier Burstable B1ms cuesta menos de $15 USD/mes. Recuerda eliminar la instancia al terminar el curso para no generar cargos.
02
PostGIS: qué agrega a PostgreSQL
+

PostGIS es una extensión que añade tipos de datos y funciones geoespaciales. Con ella, la base de datos puede almacenar y operar sobre puntos, líneas y polígonos.

  • Tipo geometry(POINT, 4326): almacena coordenadas en WGS84 (el sistema GPS)
  • ST_MakePoint(lon, lat): crea un punto desde coordenadas
  • ST_DistanceSphere: distancia en metros entre dos puntos sobre la Tierra
  • ST_Within: verificar si un punto está dentro de un polígono
  • ST_AsGeoJSON: exportar geometría como GeoJSON estándar
Activación: una sola línea SQL ejecutada una vez: CREATE EXTENSION IF NOT EXISTS postgis;
03
Azure Databricks vs. Jupyter local
+

Un Jupyter Notebook corre en tu máquina con los recursos de tu computadora. Databricks corre en un cluster de Apache Spark en la nube, distribuido en múltiples nodos.

  • Puede procesar terabytes de datos en paralelo
  • Varios usuarios colaboran en el mismo notebook en tiempo real
  • La interfaz es similar a Jupyter pero conectada a la nube
  • Acceso nativo a Blob Storage sin necesidad de descargar archivos
Para el curso: usaremos un cluster Single Node. El notebook se importa desde el archivo .ipynb que se entrega. La interfaz es prácticamente idéntica a Jupyter.
04
ABFS y JDBC: los dos puentes de Databricks
+

Databricks usa dos protocolos distintos para conectarse a las fuentes de datos del día:

  • ABFS (Azure Blob File System): lee archivos de Blob Storage directamente desde Spark, sin descargarlos a ningún disco
  • JDBC (Java Database Connectivity): protocolo estándar para conectarse a bases de datos relacionales como PostgreSQL
Importante antes del ejercicio: el servidor PostgreSQL debe tener habilitada la IP del cluster de Databricks en su firewall. El portal muestra la IP de salida del workspace en Settings > Cloud resources > Network.
