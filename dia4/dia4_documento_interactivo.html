// conceptos clave
Antes de empezar
01
¿Qué es Delta Lake y por qué no usar Parquet directamente?
+

Parquet es un formato columnar eficiente, pero no tiene transacciones. Si dos procesos escriben al mismo tiempo, los datos pueden corromperse. Delta Lake resuelve esto agregando un transaction log junto a los archivos Parquet.

  • Cada escritura genera una entrada en el log con un número de versión
  • Puedes leer el estado de la tabla en cualquier versión pasada (time travel)
  • MERGE permite hacer upserts eficientes sin leer toda la tabla
  • Compatible con Spark: se usa igual que un DataFrame normal
Analogía: Delta Lake es a Parquet lo que Git es a archivos de texto. Mismo contenido, pero con historial y control de versiones.
02
MLflow: qué es y para qué lo usamos en Databricks
+

MLflow es una plataforma open source para gestionar el ciclo de vida de modelos de ML. Databricks lo trae preinstalado y con una UI integrada.

  • Tracking: registra parámetros, métricas y artefactos de cada entrenamiento
  • Runs: cada ejecución del modelo es un "run" con su propio ID
  • Experiments: agrupa runs relacionados para comparar
  • Model registry: versiona modelos y controla su ciclo de vida
En el notebook del día: entrenaremos un modelo simple con scikit-learn y logearemos sus métricas con MLflow. La UI de Databricks permite comparar runs visualmente.
03
Azure ML Workspace: los componentes esenciales
+

Un Workspace de Azure ML es el contenedor central que agrupa todos los recursos de un proyecto de ML.

  • Compute instances: VM dedicada para desarrollo en notebooks
  • Compute clusters: clusters escalables para entrenamiento (min 0 nodos)
  • Jobs / Experiments: historial de entrenamientos integrado con MLflow
  • Environments: entornos reproducibles con dependencias de Python
  • Models: registro de modelos entrenados con versionado
Costo del día: el workspace no tiene costo propio. El costo viene del Compute Cluster solo mientras entrena. Con Standard_DS2_v2 y min 0 nodos, se apaga al terminar.
04
Command Job: cómo Azure ML ejecuta tu script de entrenamiento
+

Un Command Job es la forma principal de lanzar entrenamientos en Azure ML. Tú escribes el script, Azure ML lo empaqueta y lo ejecuta en el cluster.

  • Defines el script (main.py), el entorno Python y el compute
  • Azure ML gestiona la infraestructura, logs y artefactos automáticamente
  • MLflow loggea métricas y el modelo sin configuración adicional (autolog)
  • El modelo queda registrado y versionado en el Model Registry
Dataset del día: default_of_credit_card_clients.csv — 30,000 clientes de tarjeta de crédito. Target: predecir default del siguiente mes. Disponible en URL pública de Azure, no necesitas subirlo.
05
Databricks vs Azure ML: ¿cuándo usar cada uno?
+

Son herramientas complementarias, no competidoras. La elección depende del problema:

  • Databricks: procesamiento a escala, pipelines ETL, ML distribuido, ingeniería de datos
  • Azure ML: ciclo de vida completo de modelos, despliegue de endpoints REST, MLOps
  • Ambos juntos: Databricks procesa y prepara datos, Azure ML entrena y despliega. MLflow conecta los dos.

Crear el workspace de Azure ML

Sigue estos pasos antes de que empiece la parte de Azure ML. El workspace tarda ~2 minutos en crearse.

1
Ir al portal de Azure
Abre portal.azure.com → busca Azure Machine Learning en la barra superior → selecciona el servicio.
2
Crear nuevo workspace — clic en + Create
Subscription La del curso
Resource group El mismo que días anteriores
Workspace name workspace-curso-azure
Region La misma que tu Storage Account
Storage account El que creaste el Día 1
El resto déjalo por defecto. Clic en Review + Create → Create.
3
Abrir Azure ML Studio
Una vez creado el workspace: Go to resource → botón Launch Studio. Se abre ml.azure.com con tu workspace activo.
4
Copiar tus credenciales para el notebook
En ML Studio, haz clic en el nombre del workspace (esquina superior derecha) y copia estos tres valores:
Subscription ID xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
Resource Group nombre-resource-group
Workspace name workspace-curso-azure
5
Autenticarse con Azure CLI (en tu terminal)
El notebook usa DefaultAzureCredential, que se autentica automáticamente si iniciaste sesión con el CLI. Ejecuta en tu terminal:
az login
Se abrirá el navegador para confirmar tu cuenta de Azure. Si no tienes el CLI: https://aka.ms/installazurecliwindows
