setup_dia5.sh — un script, todo listo

El script recibe dos parámetros y crea todos los recursos en orden. Usa set -euo pipefail para detenerse ante cualquier error. Al terminar escribe un archivo credenciales.env con todos los valores para el notebook.

setup_dia5.sh
# Uso — recibe nombre del RG y zona como parámetros bash setup_dia5.sh rg-curso-dia5 eastus # ── Lo que crea el script ─────────────────────────────── [1/7] Resource Group [2/7] Storage Account + Blob container + Data Lake container [3/7] PostgreSQL Flexible Server (B1ms · PostgreSQL 16 · PostGIS) [4/7] Azure ML Workspace [5/7] Compute Cluster (Standard_DS2_v2 · min 0 nodos) [6/7] Databricks Workspace (tier Standard) [7/7] Escribe credenciales.env con todos los valores # ── Al terminar ───────────────────────────────────────── # Abre dia5_taller.ipynb y copia los valores de credenciales.env # ── Para eliminar todo al final del día ───────────────── az group delete --name rg-curso-dia5 --yes --no-wait

Buenas prácticas
01
Nunca hardcodees credenciales en Git Usa .env + .gitignore, o Azure Key Vault para producción. Una clave expuesta en GitHub puede costarte miles de dólares en minutos.
02
Etiqueta todos tus recursos --tags "proyecto=enigh" "equipo=datos" en cada recurso. Sin etiquetas, en 3 meses no sabrás qué puedes eliminar.
03
min_instances = 0 en todos los clusters Un Compute Cluster con 2 nodos inactivos cuesta ~$150 USD/mes. Con min=0 e idle_time=120s el costo cae a cero cuando no entrenas.
04
Elimina recursos al terminar proyectos az group delete --name <RG> --yes borra todo lo del grupo en una línea. Crea un RG por proyecto o taller para poder limpiar fácilmente.
05
Versiona tus datos en Blob Guarda archivos con fecha: enigh_2022_hogares_20260306.csv. O usa Delta Lake para versionado automático con time travel.
06
Documenta tus experimentos de ML MLflow guarda params y métricas automáticamente con autolog(). Sin log, en 6 meses no recordarás qué hiperparámetros usaste ni por qué elegiste ese modelo.
